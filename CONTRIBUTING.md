# Contributing to Stephen Hawking's Brain

Welcome to the collective intelligence. By contributing to this project, you are helping to expand the boundaries of our digital reconstruction of one of the greatest minds in history.

## 1. Code of Conduct

We are committed to a research environment that is respectful, scientific, and inclusive.
- Be Professional: Treat all contributors with respect.
- Stay Scientific: Base arguments on data and logic.
- Integrity: Do not submit code intended to harm the system or bias the agents.

## 2. How to Contribute

There are several ways to help the swarm evolve:
- Report Bugs: Use the issue tracker to report technical glitches or cognitive inconsistencies in the agents.
- Suggest Personas: If you believe a new specialist agent (e.g., Quantum Gravity or String Theory) should be added to the swarm, provide a detailed persona description.
- Improve UX: Help us make the Hawking Terminal more accessible and immersive.
- Code Contributions: Submit pull requests for performance optimizations or new visualization modules.

## 3. Development Workflow

To set up the lab locally:

- Clone the repository from the source.
- Install dependencies using your preferred package manager (npm install).
- Configure environment variables for Gemini and Firebase.
- Run the development server (npm run dev).

## 4. Submission Guidelines

When submitting a pull request:
- Use Descriptive Titles: Clearly state what you are changing.
- Verify Code Quality: Run the linting tool (npm run lint) before submission.
- Include Tests: If you are adding a new API or complex logic, include a test file in the src directory.
- Security Audit: Ensure your changes do not violate the Eight Pillars described in the Security Specification.

## 5. Agent Character Guidelines

If you are modifying the system prompts in gemini.ts:
- Maintain the Voice: Agents should speak with the authority and wit of an academic peer.
- Cross-Agent Awareness: New agents must be designed to interact with the existing swarm, not just provide isolated answers.
- Zero Simulation: Do not program agents to "pretend" they have external capabilities they do not have.
